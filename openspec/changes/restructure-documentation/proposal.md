## Why

The project needs to comply with a specific 10-part documentation template provided by instructors. The current documentation in `/docs` is well-structured but doesn't map directly to the required format (A-J). Restructuring will ensure all requirements are met and the project is "well presented" as requested.

## What Changes

- Create a new documentation structure in `/docs` using lowercase English names with letter prefixes (a-j).
- Use subdirectories for each section to accommodate multiple assets (images, diagrams, etc.).
- Map existing documentation from `docs/` to the new 10-section format.
- Convert or supplement the requested PDF requirements with Markdown source files.
- Update the root `README.md` to serve as the entry point for the new structure.
