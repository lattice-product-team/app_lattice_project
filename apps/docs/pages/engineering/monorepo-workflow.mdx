# Monorepo Workflow

This guide describes how to develop efficiently within the Lattice monorepo. We use **pnpm** for package management and **Docker** for local environment orchestration.

## Prerequisites

- **Node.js**: >= 18.x
- **pnpm**: >= 8.x
- **Docker & Docker Compose**

## Local Setup

1.  **Install dependencies**:
    ```bash
    pnpm install
    ```
2.  **Environment Variables**:
    Copy `.env.example` to `.env` in the root and in the respective app directories.
3.  **Start Development Environment**:
    We use Docker Compose to spin up the entire stack:
    ```bash
    docker-compose up
    ```

## Workspace Structure

Our monorepo is organized into several workspaces:

- `apps/`: Contains the primary applications (Admin Web, Mobile).
- `packages/`: Shared libraries and configurations used across the apps.
- `openspec/`: Design documents, specifications, and change management.

## Common Commands

### Development

| Command                 | Description                                              |
| :---------------------- | :------------------------------------------------------- |
| `pnpm dev`              | Starts all apps (API, Web, Docs) in development mode.    |
| `pnpm dev:api`          | Starts only the Backend API.                             |
| `pnpm dev:mobile`       | Starts the Expo dev server for the mobile app.           |
| `pnpm dev:android`      | Builds and runs the app on a physical Android device.    |
| `pnpm dev:android:prod` | Runs the mobile app connected to the **Production** API. |

### Build & Deploy

| Command                     | Description                                       |
| :-------------------------- | :------------------------------------------------ |
| `pnpm build`                | Builds all applications for production.           |
| `pnpm build:mobile:cloud`   | Generates a production build via EAS Cloud.       |
| `pnpm build:mobile:local`   | Generates a production build locally on your Mac. |
| `pnpm build:mobile:preview` | Generates an installable APK for testing.         |

### Database

| Command           | Description                               |
| :---------------- | :---------------------------------------- |
| `pnpm db:migrate` | Runs database migrations.                 |
| `pnpm db:studio`  | Opens Drizzle Studio to visualize data.   |
| `pnpm db:seed`    | Populates the database with initial data. |

## Mobile Deployment Strategy

When building the mobile app, you need to consider **where** it's built and **what** format is generated.

### Build Locations (Cloud vs. Local)

| Type      | Command Suffix | Location     | Pros                                     | Cons                    |
| :-------- | :------------- | :----------- | :--------------------------------------- | :---------------------- |
| **Cloud** | `:cloud`       | Expo Servers | No local setup needed, provides QR code. | Waiting queues, slower. |
| **Local** | `:local`       | Your MacBook | Super fast, no queues, total control.    | Uses local CPU/RAM.     |

### Build Profiles (Format & Usage)

| Profile        | Format     | Purpose           | Installation                   |
| :------------- | :--------- | :---------------- | :----------------------------- |
| **Production** | `.aab`     | App Store Release | via Google Play Store only.    |
| **Preview**    | **`.apk`** | Internal Testing  | **Direct download & install**. |

## Deployment

Deployment is automated via CI/CD pipelines. Ensure all tests pass and the build is successful locally before pushing to the main branch.
