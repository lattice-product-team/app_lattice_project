# Architectural Decision Records (ADR)

This directory contains the Architectural Decision Records (ADRs) for the Lattice project.

## What is an ADR?

An Architectural Decision Record (ADR) is a document that captures an important architectural decision made along with its context and consequences.

## Why use ADRs?

-   **Transparency**: Everyone understands *why* a decision was made.
-   **Context**: Prevents re-litigating old decisions without new context.
-   **Onboarding**: Helps new developers understand the technical history of the project.

## ADR Process

1.  **Propose**: Create a new ADR from the [template](./ADR-000-template.md) and set status to `Proposed`.
2.  **Review**: Discuss the ADR with the team.
3.  **Decide**: Set status to `Accepted`, `Rejected`, or `Deprecated`.
4.  **Implement**: Follow the decision in the codebase.

## Directory Structure

All ADRs follow the naming convention: `ADR-###-description.md`.
