# Buenas Prácticas en Desarrollo Móvil

Este documento resume las convenciones técnicas necesarias para mantener el rendimiento y la calidad en la aplicación móvil de Lattice.

## Expo y Development Builds

Este proyecto requiere **Custom Development Builds** debido al uso de módulos nativos de alto rendimiento (MMKV, Reanimated) que no son compatibles con Expo Go.

- **Comando de ejecución:** `npx expo run:android` o `npx expo run:ios`.
- **Requisito:** Tener configurado el entorno nativo local (Android Studio / Xcode).

## Integración de MapLibre GL

Utilizamos `@maplibre/maplibre-react-native` como motor de mapas principal.

### Optimización de Rendimiento
1. **SurfaceView**: Usar siempre `surfaceView={true}` en Android para evitar el overhead de renderizado.
2. **Capas Nativas (GPU)**:
   - **Evitar Marcadores Estándar**: No usar el componente `Marker` por defecto para múltiples puntos, ya que bloquea el hilo de JS.
   - **ShapeSource + Layers**: Usar `ShapeSource` combinado con `CircleLayer` o `SymbolLayer`. Esto delega el renderizado a la GPU.
   - **Estado Activo Separado**: Mantener una `ShapeSource` dedicada para el punto seleccionado para permitir actualizaciones instantáneas sin re-parsear todo el dataset.
3. **Batching**: Actualizar las formas GeoJSON mediante `useMemo` para reducir el tráfico en el puente (bridge) de React Native.

## Gestión de Estado y Animaciones

- **Estado de Servidor**: Utilizar `useQuery` para la obtención de datos, envolviendo los resultados en hooks personalizados.
- **Persistencia Local**: Usar `react-native-mmkv` para configuraciones críticas. Evitar el uso de `AsyncStorage`.
- **Animaciones**: Utilizar exclusivamente `react-native-reanimated` para todas las transiciones de UI y movimientos del mapa.

## Realidad Aumentada y Orientación

La transición entre 2D y AR está automatizada mediante sensores de orientación:

- **Portrait**: Vista estándar de mapa 2D.
- **Landscape**: El modo AR se activa automáticamente.
- **Proyección Híbrida**: En modo landscape, las etiquetas utilizan un overlay 2D (Reanimated) proyectado para mantenerse legibles independientemente del giro del sistema operativo.

## Escalabilidad y Accesibilidad

- **Componentes Atómicos**: Utilizar el patrón `SettingItem` para pantallas de configuración y perfil para asegurar consistencia visual.
- **Renderizado Condicional**: Usar siempre operadores ternarios `condicion ? <Component /> : null` para evitar renderizados inesperados de valores `0` o `NaN`.
- **Accesibilidad**: Proporcionar etiquetas de accesibilidad (`accessibilityLabel`) en todos los elementos interactivos del mapa y la interfaz.
